# Автоматическая детекция дефектов FDM 3D-печати методами глубокого обучения

Решение прикладной задачи компьютерного зрения (CV) по распознаванию брака при 3D-печати в режиме реального времени. Проект реализован на базе сверточных нейронных сетей (CNN) с использованием фреймворка **PyTorch**.

##  Актуальность проекта
Контроль качества FDM-печати вручную неэффективен при масштабировании производства. Данное решение позволяет автоматизировать детекцию брака на ранних стадиях, минимизируя порчу расходных материалов (пластика) и предотвращая повреждения оборудования.

##  Описание и подготовка данных
В проекте используется датасет, содержащий **1912 изображений**, разделенных на 5 классов (4 типа дефектов + нормальное состояние):
* **Cracking** (трещины/расслоение) — 472
* **Layer shifting** (сдвиг слоев) — 364
* **Off platform** (отрыв детали от стола) — 91 *(критический дисбаланс)*
* **Stringing** (паутина/волосатость пластика) — 447
* **Warping** (деформация/загибание углов) — 538

### Предобработка и аугментация (Pipeline)
Для борьбы с переобучением и улучшения обобщающей способности модели применен комплекс методов аугментации данных (`torchvision.transforms`):
* Изменение размера до $224 \times 224$ пикселей и нормализация под стандарты ImageNet.
* Случайные горизонтальные и вертикальные перевороты (`RandomHorizontalFlip`, `RandomVerticalFlip`).
* Случайные повороты на угол до 15 градусов (`RandomRotation`).
* Цветовой джиттеринг (`ColorJitter`) — изменение яркости, контрастности и насыщенности для симуляции разного освещения в камере 3D-принтера.

### Разделение выборки:
* **Train (Обучающая):** 1338 изображений (70%)
* **Validation (Валидационная):** 287 изображений (15%)
* **Test (Тестовая):** 287 изображений (15%)

##  Архитектура модели
В качестве базовой архитектуры использована кастомная глубокая сверточная нейросеть (CNN):
* **Feature Extractor:** Блоки из сверточных слоев `Conv2d` с увеличением количества каналов, слои пакетной нормализации `BatchNorm2d` (для стабилизации обучения) и функции активации `ReLU`.
* **Pooling:** Слои `MaxPool2d` для снижения размерности признаков.
* **Classifier:** Полносвязные слои (`Linear`) с применением `Dropout` (p=0.5) для предотвращения коадаптации признаков и финальным слоем на 5 выходов.
* **Функция потерь:** Многоклассовая кросс-энтропия (`CrossEntropyLoss`).
* **Оптимизатор:** `Adam` с начальным шагом обучения (learning rate) $0.001$.

##  Процесс обучения и результаты
Модель обучалась на протяжении **10 эпох** с использованием графического ускорителя (CUDA). 

* **Итоговая точность на валидационной выборке (Validation Accuracy):** **72.82%**
* Наблюдается стабильное снижение функции потерь (Loss) как на обучающей, так и на валидационной выборке, что свидетельствует о корректной сходимости градиентного спуска.

### Анализ метрик и ошибок:
Модель показывает высокую точность на классах с характерными геометрическими признаками (*Stringing*, *Warping*). Небольшое проседание метрик в классе *Off platform* обусловлено сильным изначальным дисбалансом данных (всего 91 изображение в датасете), что является классическим вызовом для систем компьютерного зрения и точкой роста для дальнейшей настройки весов функции потерь (Weighted CrossEntropy).
